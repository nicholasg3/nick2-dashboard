3. Root Cause

PMO cannot scale triage without a shared rubric — ad-hoc ranking would misallocate the $20/week cap across low-ROI agent runs.

4. Strategic Options

Option A: Approve default framework (pilot)

  • Upside: Unblocks PMO-001 immediately; impact×readiness/effort is good enough for first cycle.
  • Downside: Weights may need tuning after first spend report.
  • When to choose: Nick wants velocity over perfect calibration.

Option B: Approve with Nick-tuned weights

  • Upside: Better fit to Nick's ROI preferences before any spend.
  • Downside: Adds one review round; delays first dispatch.
  • When to choose: Nick has strong priors on issue priority.

Option C: Defer — manual CEO triage only

  • Upside: Zero framework risk.
  • Downside: Does not scale; wastes PMO capacity; blocks autonomous loop.
  • When to choose: Only if autonomous dispatch is off indefinitely.

Option D: Reject — revert to ad-hoc picks

  • Upside: No process overhead.
  • Downside: Highest risk of budget waste and inconsistent outcomes.
  • When to choose: Not recommended while budget is capped and worker may enable.

5. Recommendation

Approve the default framework for a pilot on the top 3 issues. Revisit weights after the first weekly spend report (CFO).
