# POL-001: Nick gate queue convention — Completed

**Owner:** CEO · **Status:** completed · **Updated:** 2026-06-30T16:22 · **Cost:** $0.00


## 1. Executive Framing

**Objective:** Prevent agent idle time when work is gated on Nick.  
**Outcome:** If work needs Nicholas, append nick_gate to ledger and memos/gated/. Agents keep executing ungated Active Work Queue items — never idle on Nick.

## 2. What shipped (MECE)

| Bucket | Scope | Current state |
|--------|-------|---------------|
| Gate | nick_gate / decision_needed | Park in Gated queue |
| Execute | Active Work Queue | Ungated only |
| Clear | nick_gate_resolved | Nick or reconcile |
| Anti-pattern | Blocking on Nick | Explicitly forbidden |

## 3. Root cause addressed

Without a gate convention, agents stall on human dependencies.

## 5. Recommendation

**Policy active.** Agents continue ungated work while Nick clears gates.
